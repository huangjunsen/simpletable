2. 使用方法：

  • 2.1. 添加CSS及JS文件
    <link href="static/plugins/simpletable/simpletable.min.css" rel="stylesheet" type="text/css" />
    <script src="static/plugins/simpletable/simpletable.min.js" type="text/javascript"></script>
  • 2.2. 书写 Table Dom
    <table id="SimpleTable">
      <thead>
        <tr><th>...</th></tr>
      </thead>
      <tbody>
        <tr><td>...</td></tr>
        <tr><td>...</td></tr>
      </tbody>
      <tfoot>
        <tr><th>...</th></tr>
      </tfoot>
    </table>
  • 2.3. 采用js方式生成：
    $('#SimpleTable').SimpleTable(options);

3. 非js方式生成：

  • 3.1 朴素 SimpleTable ： 为 table 添加 .simpletable 类即可。
  • 3.2 激活状态： 为 tr 添加 .tr-active 类即可产生改行的激活状态。
  • 3.3 禁用状态： 为 tr 添加 .tr-disable 类即可产生改行的禁用状态。
  • 3.4 6种主题： 为 .simpletable 类添加 .tab-* 即可产生5种色彩主题（*可以为primary/success/info/warning/danger）。
  • 3.5 无边框样式：为 .simpletable 类添加 .noborder 类可以产生无边框的表格效果。
  • 3.6 极其简单的Dom： table 中可以仅仅含有 tr th td 等元素而不必含有 thead tbody tfoot 等元素（尽管推荐使用这些标签）。
  • 3.7 对齐状态：为 .simpletable 类添加 .text-* 即可不同的对齐状态（*可以为left/center/right）。
  • 3.8 响应式表格：为表格外层添加一个div，并附上 .table-responsive 类，可以达到响应式的效果（依赖bootstrap）。

5. js方法：

采用纯粹 Html 方式虽然能展现出效果不错的表格，但是一些需要js支持的功能却无法使用，这是一种遗憾。
因此在插件中添加几个较为有用的方法，提供使用：

方法示例描述
addHover 
.SimpleTable('addHover');
 为表格添加鼠标经过样式
addColumn 
.SimpleTable('addColumn',options);
 批量为表添加列，参数options介绍（以下为默认值）：
{
  colCount:1,    //需要添加的列数
  contents: [],  //嵌套数组，用于存放需要添加的内容
  hasID: false,  //是否需要为每个单元格添加id属性
  headIDPrefix: 'H', //th的id前缀
  lineIDPrefix: 'L', //td的id前缀
  startID: 0     //id的起始序号
}
rowspan 
.SimpleTable('rowspan','all');.SimpleTable('rowspan',0);
 合并内容相同的单元格为跨行单元格：'all'/行号(起始为0)
colspan 
.SimpleTable('colspan','all');.SimpleTable('colspan',0);
 合并内容相同的单元格为跨行单元格：'all'/列号(起始为0)
rowactive 
.SimpleTable('rowactive','all');.SimpleTable('rowactive',0);
 将全部行/制定行置为禁用状态：'all'/行号(起始为0)
注意，对标题行不能生效。
rowdisable 
.SimpleTable('rowdisable','all');.SimpleTable('rowdisable',0);
 将全部行/制定行置为禁用状态：'all'/行号(起始为0)

Code: 
$('#SimpleTable3').SimpleTable('rowspan','all');
$('#SimpleTable3').SimpleTable('colspan','all');

特别需要注意以下内容：
1. 由于“被合并”的单元格并非实际删除，而仅仅只是隐藏了（加上了display:none;），因此每一行的数据依然还是存在的，在进行单元格操作的时候注意别忘了这个单元格。
2. 一般情况下只会存在跨行或者跨列的情况，不会存在既跨行又跨列的情况，这个在Html中是没有办法通过简单方式实现的，如果存在跨越多行及多列的情况，切记一定要保证所有跨越的行和列都是同一个值（如例子中的TD4.3），否则将产生不可预知的错位。
3. th 和 td 之间不能跨越，所以请保证数据首行（或者尾行）与 标题行没有相同的单元格。
